# JSON/PDF Checker

JSON/PDF Checker は、論文 PDF を見ながら著者と所属機関の情報を確認・修正するためのツールです。

アノテーターは、JSON または JSONL ファイルを読み込み、1 件ずつ論文を確認します。PDF に書かれている内容と JSON の内容を比べて、著者名、メールアドレス、ORCID、所属機関などを修正し、最後に修正済み JSONL をダウンロードします。

## まず覚えること

- PDF に書かれている情報を優先します。
- 分からない情報を推測で追加しないでください。
- 判断に迷う場合は「管理者によるチェックが必要」にチェックを入れます。
- 作業後は必ず「修正済みJSONLをダウンロード」してください。

## セットアップ

この手順は、ターミナルで `linking_checker` ディレクトリに移動してから実行します。

```bash
python -m venv .venv
```

仮想環境を有効にします。

macOS / Linux:

```bash
. .venv/bin/activate
```

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

必要なパッケージをインストールします。

```bash
pip install -r requirements.txt
```

## 起動方法

次のコマンドを実行します。

```bash
streamlit run app.py
```

ブラウザでアプリが開きます。開かない場合は、ターミナルに表示された URL をブラウザに貼り付けてください。

## 入力ファイル

読み込めるファイルは `.json` または `.jsonl` です。

JSONL は、1 行に 1 論文の JSON が入っている形式です。このツールでは、主に次のような情報を確認します。

```json
{
  "arxiv_id": "2603.03108v1",
  "doc_class": "article",
  "authors": [
    {
      "name": "Author Name",
      "email": "",
      "orcid": "",
      "other": {},
      "affiliations": [
        {
          "institution": "University Name",
          "address": ""
        }
      ]
    }
  ]
}
```

`arxiv_id` が入っている場合、PDF URL は自動で `https://arxiv.org/pdf/<arxiv_id>` になります。

## 基本的な作業手順

1. 左側のサイドバーで「JSON / JSONL」から作業ファイルをアップロードします。
2. 1 件目の論文が表示されます。
3. PDF を開いて、論文の著者・所属情報を確認します。
4. 「フォーム編集」タブで JSON の内容を確認・修正します。
5. 必要に応じて「差分」タブで、読み込み時の内容と現在の内容を比べます。
6. 判断に迷う場合は「管理者によるチェックが必要」にチェックを入れ、理由を書きます。
7. 「確認して次へ」を押します。
8. 内容確認の画面が出たら、問題なければもう一度「確認して次へ」を押します。
9. すべての論文を確認したら、左側の「修正済みJSONLをダウンロード」を押します。

## 画面の見方

### サイドバー

左側のサイドバーでは、ファイルの読み込み、論文の移動、PDF URL の確認、修正済み JSONL のダウンロードを行います。

- 「JSON / JSONL」: 作業するファイルをアップロードします。
- 「修正済みJSONLをダウンロード」: 現在の作業結果を保存します。
- 「前の論文」「次の論文」: 前後のレコードに移動します。
- 「n番目の論文へ」: 指定した番号の論文に移動します。
- 「確認するレコード」: 一覧から論文を選びます。
- 「PDF URL」: 表示する PDF の URL です。通常は自動で入ります。

### フォーム編集

通常の修正は「フォーム編集」タブで行います。

- `arxiv_id`: arXiv の ID です。編集できません。
- `doc_class`: TeX の document class です。編集できません。
- `Name`: 著者名です。
- `Email`: メールアドレスです。
- `ORCID`: ORCID です。
- `その他`: 追加情報がある場合に使います。
- `Affiliation`: 著者の所属情報です。
- `Institution`: 所属機関名です。

著者が足りない場合は「著者を追加」を押します。所属が足りない場合は、該当する著者の中で「所属を追加」を押します。不要な著者や所属は削除できます。

### Raw JSON

「Raw JSON」タブでは、選択中のレコードを JSON として直接編集できます。

初心者は、まず「フォーム編集」を使ってください。Raw JSON は、フォームでは編集しにくい項目を直す必要がある場合だけ使います。

Raw JSON を編集したら「Raw JSONを反映」を押します。JSON の書き方に間違いがあるとエラーになります。

### 差分

「差分」タブでは、左に読み込み時の JSON、右に現在の JSON が表示されます。

修正した内容が正しいか確認したいときに使います。何も変更していない場合は、「このレコードはまだ変更されていません。」と表示されます。

### 確認ステータス

画面下部の「確認ステータス」では、そのレコードの確認状態を管理します。

- 何も問題がなければ、そのまま「確認して次へ」を押します。
- 判断に迷う場合は「管理者によるチェックが必要」にチェックを入れます。
- チェックを入れた場合は、理由を短く書きます。

ステータスは自動で設定されます。

- `confirmed`: 内容を確認し、修正がなかったレコード
- `modified`: 内容を修正したレコード
- `needs_review`: 管理者チェックが必要なレコード
- `unconfirmed`: まだ確認が完了していないレコード

## アノテーションの基本ルール

PDF の著者欄、所属欄、脚注、メールアドレス表示などを確認します。

著者名は、PDF に書かれている表記に合わせます。省略や別表記がある場合は、PDF 上で正式に見える表記を使います。

所属機関は、著者と所属の対応関係が分かるように入力します。著者に複数の所属がある場合は、その著者に複数の Affiliation を追加します。

メールアドレスや ORCID は、PDF 上で確認できる場合だけ入力します。確認できない場合は空欄のままで構いません。

所属の住所や国名が必要か迷う場合は、既存データの書き方に合わせます。判断できない場合は管理者チェックに回してください。

## 迷ったとき

次のような場合は「管理者によるチェックが必要」にチェックを入れてください。

- 著者と所属の対応関係がはっきり分からない。
- 同じ著者に複数の所属があり、どこまで入力するか迷う。
- PDF の表示と JSON の内容が大きく違う。
- メールアドレスや ORCID が誰のものか判断できない。
- PDF が開けず、確認できない。
- JSON の構造が崩れているように見える。

理由欄には、何に迷ったかを短く書きます。

例:

```text
著者2と著者3の所属番号がPDF上で判別できない。
```

## 保存について

このツールは、作業結果を自動でファイル保存しません。

作業が終わったら、必ず左側の「修正済みJSONLをダウンロード」を押してください。ダウンロードされるファイル名は `review_result.jsonl` です。

途中でブラウザを閉じたり、ページを更新したりすると、未保存の作業内容が失われる場合があります。長時間作業する場合は、こまめにダウンロードしてください。

## よくある困りごと

### PDF が開かない

まず PDF URL が正しいか確認してください。`arxiv_id` が空の場合、PDF URL は自動生成されません。

ブラウザやネットワークの状態によって、PDF が表示されないことがあります。その場合は、PDF URL をコピーして別のタブで開いてください。確認できない場合は、管理者チェックに回してください。

### JSON / JSONL が読み込めない

ファイル形式が `.json` または `.jsonl` か確認してください。

JSONL の場合は、1 行に 1 つの JSON object が入っている必要があります。途中に壊れた行があると読み込みに失敗します。

### Raw JSON を反映できない

JSON の書き方に間違いがある可能性があります。

よくある間違い:

- カンマが足りない。
- 余分なカンマがある。
- `"` が閉じていない。
- `{}` や `[]` の対応が崩れている。

初心者は Raw JSON を直接編集せず、できるだけ「フォーム編集」を使ってください。

### 次の論文に進む前に確認したい

「差分」タブを開いて、修正前と修正後を比べてください。その後、「確認して次へ」を押します。

## 補足

`main.py` は古い確認用スクリプトです。通常のアノテーション作業では使用しません。

作業には `app.py` の Streamlit アプリを使ってください。
